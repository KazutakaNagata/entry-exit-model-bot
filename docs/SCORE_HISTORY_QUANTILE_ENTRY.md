# Score-history quantile entry for monthly family ablation

`rolling_quantile` was useful while the model was fixed and the validation period
was long.  In monthly rolling research, each validation/test month already has a
fresh model trained on prior data, so a simpler live-safe entry rule is often
cleaner:

```text
threshold = quantile(pre_period_score_history, q)
entry if current_score > max(threshold, floor_bps)
```

In this repository this mode is named:

```yaml
entry_selection:
  mode: score_history_quantile
  score_quantile: 0.995
  score_floor_bps: 0.0
  score_quantile_min_periods: 30000
```

The `score_history` file is generated by the monthly model for the period before
the validation/test month.  With the default monthly config, this is the previous
120 days.  The evaluation-month score distribution is not used.

## Run a new monthly family-ablation batch with score-history quantile

```bash
python3 scripts/prepare_rolling_monthly_family_ablation_jobs.py \
  --config configs/rolling_protocol/long_H120_monthly_family_ablation_score_history_quantile_v0.yaml \
  --first-test-month 2025-04-01 \
  --last-test-month 2025-07-01 \
  --batch-id long_H120_family_ablation_scoreQ_202504_202507
```

```bash
bash outputs/valid/rolling_monthly_family_ablation_jobs/long_H120_family_ablation_scoreQ_202504_202507/run_all_parallel.sh 4
```

or with tmux:

```bash
bash outputs/valid/rolling_monthly_family_ablation_jobs/long_H120_family_ablation_scoreQ_202504_202507/launch_tmux.sh long_h120_scoreq
```

## Replay an old monthly run without retraining

Existing rolling-quantile monthly runs contain `entry_oof_predictions.parquet` and
`score_history.parquet` for every validation candidate.  You can replay the valid
candidate selection with score-history quantile without retraining:

```bash
python3 scripts/replay_monthly_family_ablation_score_quantile.py \
  --source-run-name long_H120_family_ablation_202504_202507_202505 \
  --run-id long_H120_family_ablation_202505_scoreQ_replay \
  --score-quantile 0.995 \
  --score-floor-bps 0
```

Caveat: old runs usually only have test predictions for the policy selected by
the old selector.  If the replay chooses a different policy, the script reports
`missing_selected_test_predictions`; it does not infer or fabricate test P/L.
