# Rolling History Modes

`rolling_quantile` entry selection must not use a full validation fold's future score distribution. This project now supports three live-safe history modes for the rolling score threshold.

## Modes

### `fold_local`

Default, previous behavior.

For each valid fold, the rolling quantile is computed from rows earlier than the current timestamp within the same fold only.

```text
fold_01: fold_01 past scores only
fold_02: fold_02 past scores only
```

This is leak-safe but conservative: fold starts have no warmup history, so entries are blocked until `rolling_score_min_periods` past scores exist.

### `continuous_valid`

Treats valid OOF predictions as one chronological stream.

```text
fold_01: fold_01 past scores only
fold_02: fold_01 + fold_02 past scores
fold_03: fold_01 + fold_02 + fold_03 past scores
```

This still excludes current and future rows via `closed='left'`. It is useful for testing 90d / 120d rolling windows when each fold is only about three months.

Caveat: because OOF folds may be produced by different fold models, previous-fold score distributions can have mild scale differences. Use this as a practical live-style approximation. For final candidates, compare with explicit `score_history` generated by the same evaluation fold model.

### `score_history`

Uses an explicit score-history file for rolling warmup. This is intended for train-tail or pre-fold history.

Expected columns:

```text
timestamp
side
horizon_minutes
pred_entry_net_bps
fold  # optional
```

If `fold` is present, rows tagged with the current fold are used for that fold. Shared rows may use `*`, `all`, `global`, or `train`.

## Commands

Continuous valid rolling window sweep:

```bash
python3 scripts/sweep_selected_long_h120_rolling_windows.py \
  --rolling-history-mode continuous_valid \
  --window-days 60,90,120 \
  --quantiles 0.995 \
  --floors 40,50
```

Single fixed-hold run:

```bash
python3 scripts/backtest_fixed_hold_entry_valid.py \
  --ohlcv data/processed/binance_japan/BTCJPY/1m/ohlcv.parquet \
  --entry-oof outputs/valid/entry_oof/selected_long_H120_long_H120_tail_v0/entry_oof_predictions.parquet \
  --side long \
  --entry-horizon 120 \
  --fixed-hold-minutes 120 \
  --entry-selection-mode rolling_quantile \
  --rolling-history-mode continuous_valid \
  --rolling-score-window-days 120 \
  --rolling-score-quantile 0.995 \
  --rolling-score-min-periods 30000 \
  --entry-score-floor-bps 40 \
  --entry-threshold-bps 40 \
  --roundtrip-cost-bps 15
```

## Interpretation

`fold_local` answers: "Can the strategy work if each fold starts cold?"

`continuous_valid` answers: "What if live score history is carried forward through valid time?"

`score_history` answers: "What if each fold starts with explicitly generated pre-fold score history?"

Do not use fold-wide top quantiles as a backtest rule.
